**任务：** 修正右手臂关节坐标系方向异常问题

## 问题描述

在V11机器人的SMPLX到机器人关节映射配置中，右手臂关节（`right_elbow_link`和`right_wrist_yaw_link`）的坐标系方向存在异常：

- **问题表现：** 在实际可视化中观察到坐标系方向为"x朝右上，y朝左上，z朝下"
- **期望状态：** 与其他关节一致的"标准右手坐标系"（X朝前，Y朝右，Z朝上）
- **影响：** 导致右手臂运动控制异常，与其他关节方向不一致

---

## 问题根源分析

通过深入分析，发现问题的根本原因：

### 1. URDF关节定义分析
- `right_elbow_joint`在URDF中定义了87度Y轴旋转：`rpy="0 1.52 0"`
- 这个旋转在当前的配置校正中没有被正确处理

### 2. 数学验证
- 左臂配置正常：`[0.8, -0.331, -0.462, 0.191]`
- 当前右手臂配置异常：`[0.800, 0.331, -0.462, -0.191]`
- 两个配置在数学上不满足对称性关系

### 3. 坐标系分析
- 左臂坐标系：X轴朝前+右+上，Y轴朝右+下，Z轴朝后+右+上
- 右臂期望：应与左臂坐标系方向一致（考虑镜像对称）

---

## 解决方案

### 核心思路
计算一个校正四元数，使得：
```
新配置 × URDF_87度旋转 = 左臂配置
```

### 数学推导
1. 左臂配置：`q_left = [0.8, -0.331, -0.462, 0.191]`
2. URDF 87度Y轴旋转：`q_urdf = [cos(1.52/2), 0, sin(1.52/2), 0]`
3. 新配置计算：`q_new = q_left × q_urdf⁻¹`

### 计算过程
- URDF旋转角度：87度 = 1.52弧度
- URDF旋转四元数：`[0.724836, 0, 0.688921, 0]`
- URDF旋转的逆：`[0.724836, 0, -0.688921, 0]`
- 新配置 = 左臂配置 × URDF旋转的逆

### 计算结果
```
新配置四元数：q_new = [0.261587, -0.108337, -0.886011, 0.366477]
```

## 关键发现

### 1. URDF旋转的重要性
- V11机器人URDF中的`right_elbow_joint`定义了87度Y轴旋转
- 这个旋转必须在映射配置中正确补偿
- 之前的配置忽略了这一旋转，导致坐标系方向异常

### 2. 对称性关系
- 右臂配置不是简单的左臂配置镜像
- 需要额外补偿URDF中的关节旋转
- 正确的对称关系：`q_right = q_left × q_urdf⁻¹`

### 3. 配置一致性
- `right_elbow_link`和`right_wrist_yaw_link`应使用相同的四元数校正
- 确保整个右手臂的坐标系方向一致性
